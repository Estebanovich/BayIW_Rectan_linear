# BayIW_Rectan_linear

MITgcm numerical simulation experiment to study **internal wave generation and wind-driven coastal circulation in an idealized rectangular bay domain**. The experiment compares ocean response with and without a bay to isolate how bay topography modulates internal wave dynamics.

## Scientific context

The model simulates a coastal ocean with an idealized parabolic bay:

```
h(r) = h₁ · [1 - (r² / L²)]
```

where the bay is ~119 km wide and ~164 m deep, connected to an open ocean shelf reaching ~1000 m depth. Wind forcing is applied meridionally, and the model tracks the resulting internal wave field and circulation patterns.

Two parallel runs are configured:

| Run | Description |
|-----|-------------|
| `run_expand/` | Full domain **with bay** |
| `run_expand_nobay/` | Same domain **without bay** (flat shelf) |

Comparing both runs isolates the dynamical role of the bay in internal wave generation.

## Model configuration

| Parameter | Value |
|-----------|-------|
| Model | MITgcm (non-hydrostatic capable) |
| Grid | Cartesian, 560 × 352 × 50 (Nx × Ny × Nr) |
| Horizontal resolution | Variable (non-uniform dx, dy via binary files) |
| Vertical levels | 50 (1 m near surface, up to ~46 m at depth) |
| Time step | 30 s |
| Equation of state | Linear (`eosType='LINEAR'`, temperature only) |
| Coriolis parameter | f₀ = 6.97 × 10⁻⁵ s⁻¹ |
| Horizontal viscosity | νₕ = 100 m² s⁻¹ |
| Vertical viscosity | νᵣ = 10⁻⁵ m² s⁻¹ |
| Free surface | Implicit |
| Open boundaries | Orlanski radiation (S, W, E) + sponge layers |

### MITgcm packages

- `gfd` — core geophysical fluid dynamics
- `obcs` — open boundary conditions (Orlanski radiation, sponge relaxation)
- `diagnostics` — flexible output diagnostics
- `mnc` — NetCDF output via MNC

### Open boundary conditions

- **South, West, East**: Orlanski radiation scheme
- **Sponge layer**: 10-grid-point relaxation zone (90–600 s timescales)
- **Balance**: mass-flux correction applied to maintain volume conservation

## How to run

### 1. Compile

```bash
bash compile_and_run_expand.sh
```

The script will interactively ask whether to:
- Clean the `build/` directory before compiling
- Enable MPI compilation
- Clean and run both `run_expand/` and `run_expand_nobay/`

The build uses `genmake2` with the `darwin_arm64_gfortran` optfile (Apple Silicon with gfortran).

### 2. Run individual experiments

```bash
cd run_expand/
bash run_expand.sh
```

Output is written to `output.txt`. NetCDF files are written to `mnc_000*/` subdirectories.

### 3. Post-process

Use the Jupyter notebooks in `input/` to inspect and visualize output:

```bash
jupyter notebook input/check_output_expand.ipynb
```

## Input binary files

Binary input files (64-bit big-endian) are generated by the notebooks and scripts in `input/`:

| File | Description |
|------|-------------|
| `bahia_01_expand_bat.bin` | Bathymetry |
| `bahia_01_expand_dx.bin` | Zonal grid spacing |
| `bahia_01_expand_dy.bin` | Meridional grid spacing |
| `merid_local_bay_560x352.bin` | Meridional wind stress forcing |
| `linear_temp_50zlev_560x352.bin` | Initial temperature profile |

## Dependencies

- [MITgcm](https://mitgcm.org/) — compiled separately; the `build/` directory expects `genmake2` at `../../../tools/genmake2`
- gfortran (tested on Apple Silicon with `darwin_arm64_gfortran` optfile)
- OpenMPI (optional, for parallel runs)
- Python 3 with: `numpy`, `scipy`, `matplotlib`, `xarray`, `netCDF4`
- Jupyter Notebook (for pre/post-processing)
